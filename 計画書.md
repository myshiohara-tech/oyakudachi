# お役立ちフレーズ学習アプリ 計画書

## 1. プロジェクト概要

**目的**：英会話スクールの生徒が「お役立ちフレーズ」を効率的に暗記・定着できるWebアプリを構築し、Netlifyでデプロイする。

**コンセプト**：日本語 → 英訳のアウトプット練習を、テキスト入力・音声入力・耳での反復という複数チャネルで行えるトレーニングシステム。

## 3. 機能仕様

### 3-1. 共通機能
- **カテゴリ選択画面**：上記9カテゴリから学習対象を選択
- **モード選択**：テキストモード／音声モード
- **進捗管理**：localStorageで「正解／不正解」「学習回数」を記録
- **シャッフル再生**：ランダム出題オプション

### 3-2. テキストモード
1. 日本語フレーズを画面に表示
2. ユーザーは以下のいずれかで回答：
   - **キーボード入力**（テキストエリア）
   - **音声入力**（Web Speech API の SpeechRecognition）→ 自動で文字起こし
3. 「答え合わせ」ボタンで正解判定
   - 完全一致／類似度判定（大文字小文字・句読点を無視、レーベンシュタイン距離など）
   - 不正解時は差分をハイライト表示
4. 正解の英訳とTTS音声再生（Web Speech API SpeechSynthesis）

### 3-3. 音声モード（添付m4aの形式を参考）
1. 「日本語音声」を自動再生
2. **無音ポーズ**（心の中で音読する時間／秒数は調整可：デフォルト3〜5秒）
3. 「英訳音声」を自動再生
4. 次のフレーズへ自動進行（連続再生／ステップ再生切替可）
5. **再生速度調整**（0.75x / 1.0x / 1.25x）

## 4. 技術スタック

| 領域 | 採用技術 | 選定理由 |
|---|---|---|
| フレームワーク | **Vite + React + TypeScript** | Netlify相性◎、SPAで完結 |
| スタイリング | Tailwind CSS | 高速プロトタイピング |
| 音声認識 | Web Speech API (`SpeechRecognition`) | 外部API不要・無料 |
| 音声合成 | Web Speech API (`SpeechSynthesis`) ＋ 既存m4a | TTSは即時生成、品質重視時はm4aを利用 |
| データ管理 | JSON（静的ファイル） | PDFから事前抽出、ビルド時に同梱 |
| ホスティング | **Netlify** | 静的SPAデプロイ、独自ドメイン対応、PWA化容易 |
| 永続化 | localStorage | サーバーレス維持、無料運用 |

## 5. データ準備フロー（事前処理）

1. **PDF → JSON抽出スクリプト**を作成（Node.js + pdf-parse）
   - 9つのPDFをパースし、以下の形式に変換：
   ```json
   {
     "category": "1",
     "title": "主導権を握るための英語フレーズ(導入パート)",
     "phrases": [
       { "id": 1, "ja": "私の声が聞こえて顔が見えますか？", "en": "Can you hear and see me?" }
     ]
   }
   ```
2. 抽出結果を目視チェック（番号ずれ・改行ミス対応）
3. `/public/data/` 配下に配置
4. **音声モード用の分割**：既存m4aは「日本語＋英訳」が連続している形式なので、必要に応じて
   - (A) m4aをそのまま使い、タイムスタンプで頭出し再生
   - (B) フレーズ単位でTTSをブラウザ生成（最も簡便）
   - (C) 後日、フレーズごとに分割した音声ファイルを準備

→ **MVPでは(B)のWeb Speech API TTS**を採用、将来(C)へアップグレード可能な設計に。

## 6. 画面構成（UI設計）

```
[トップ] カテゴリ一覧（9カード）
   └─ [カテゴリ詳細] モード選択（テキスト / 音声）
        ├─ [テキストモード] 出題 → 回答（キーボード/マイク）→ 判定 → 次へ
        └─ [音声モード]    自動再生（日本語 → ポーズ → 英訳 → 次へ）
[設定]   ポーズ秒数・再生速度・シャッフル・進捗リセット
[進捗]   カテゴリ別の正答率・学習回数
```

## 9. 確認事項

着手前に以下をご判断いただきたいです：

1. **音声モードの音声**：MVPはブラウザTTSで進めてよいか？（より自然な音声が必要なら、フレーズ単位の録音 or ElevenLabs等のAPI利用を検討）
2. **音声入力での正誤判定**：表記揺れ（"I'm" vs "I am"）は許容するか、厳密一致か？
3. **デザインの方向性**：シンプル教材風／親しみやすいポップ調／スクールのブランドカラー反映 等
4. **アプリ名・URL**：Netlifyサブドメイン名（例：`yoshi-phrases.netlify.app`）または独自ドメイン
