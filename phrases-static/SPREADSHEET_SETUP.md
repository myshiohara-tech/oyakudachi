# スプレッドシート連携 セットアップ手順

生徒さんごとの学習ログを Google スプレッドシートに自動記録します。
**生徒1人につき1つの専用シート**が自動で作成され、進捗・正答率・つまずきフレーズが集計されます。

所要時間: **5〜10分**（一度だけ）

---

## 1. スプレッドシートを新規作成

1. <https://sheets.new> を開く（Google アカウントでログイン）
2. ファイル名を任意に変更（例: `お役立ちフレーズ_学習ログ`）

---

## 2. Google Apps Script を貼り付ける

1. メニューから **拡張機能 → Apps Script** を開く
2. デフォルトの `Code.gs` の中身をすべて削除
3. このリポジトリの [google-apps-script/Code.gs](google-apps-script/Code.gs) の内容をコピーして貼り付け
4. 💾 アイコンで保存（プロジェクト名は任意。例: `phrases-app-logger`）

---

## 3. Web App としてデプロイ

1. 右上の **デプロイ → 新しいデプロイ** をクリック
2. ⚙️ アイコン → **ウェブアプリ** を選択
3. 設定:
   - **次のユーザーとして実行**: `自分`
   - **アクセスできるユーザー**: `全員`（記録だけなのでこれでOK。生徒のGoogleログイン不要）
4. **デプロイ** をクリック
5. 初回は権限の承認ダイアログ → 自分のGoogleアカウントで承認
   - 「このアプリは確認されていません」と出たら、**詳細 → (プロジェクト名) に移動 (安全ではないページ) → 許可**
6. 表示される **ウェブアプリの URL**（`https://script.google.com/macros/s/.../exec`）をコピー

> 🔁 後でスクリプトを編集した場合は **デプロイの管理 → 編集 → 新しいバージョン** で更新してください。

---

## 4. アプリにURLを設定

アプリの設定は管理者（講師）だけが行います。

1. お役立ちフレーズアプリを開く
2. **適当なメアド・氏名でログイン**（管理者用でOK）
3. 右上の **⚙ 設定** → **記録先（管理者向け）** に上でコピーしたURLを貼り付け
4. **🔌 テスト送信** ボタンで動作確認 → スプレッドシートに `connection_test` 行が記録されればOK ✅

> 💡 各生徒の端末でも同じURLを設定する必要があります。生徒が複数の端末を使う場合や、アプリをNetlifyに公開して使う場合は、URLをアプリの初期設定に組み込む（[コードを編集](app.js)して `defaultSettings.sheetEndpoint` に直書き）と、生徒側で設定不要になります。

---

## 5. 生徒さんがアプリを使う

1. 生徒がアプリを開くと **ログイン画面** が表示されます
2. **メールアドレス**と**氏名**を入力してログイン
   - 同じ端末では一度ログインすれば次回から自動ログインされます
   - 別の生徒が使う場合は、設定画面から「ログアウト」してから再ログイン
3. 学習を始めると、ログが自動でスプレッドシートに送信されます

---

## 6. スプレッドシートで進捗を見る

生徒が学習を始めると、自動で以下のシートが作成・更新されます。

### 📋 ログ シート（マスター）
全生徒・全アクションの生データ。
- 列: timestamp / sessionId / **email** / 氏名 / mode / action / categoryId / phraseId / 日本語 / 英訳 / 入力値 / 正解 / 類似度 / receivedAt

### 👤 生徒シート（1人につき1枚、自動作成）
シート名は生徒の氏名（同名の場合は `氏名 (2)` など）。
A1〜A2 にプロフィール（氏名・メアド）、A4以降に集計が入ります。

メニューから **📊 学習レポート → サマリーを再集計** を実行すると、以下が更新されます：

- **学習サマリー**（A4〜B11）
  - 初回学習 / 最終学習 / 学習セッション数
  - テキスト挑戦回数 / 正解回数 / 総合正答率
  - 音声モード再生フレーズ数
- **カテゴリ別正答率**（D4以降）
  - カテゴリごとの 挑戦・正解・正答率（%）
- **つまずきフレーズ Top 10**（D15以降）
  - 不正解が多いフレーズを優先的にリスト化（復習に使える）
- **学習ログ**（30行目以降に追記され続ける生ログ）

### 📊 サマリー シート（横断ビュー）
全生徒を1行ずつ並べ、総合正答率・カテゴリ別正答率を一覧化。
- 列: 氏名 / メアド / シート名 / 総挑戦 / 総正解 / 総合正答率 / セッション数 / 最終学習日時 / カテゴリごとの正答率 …

### `_students` シート（隠しシート）
メアド⇄シート名のマッピング索引。**触らないでください**（誤って編集すると生徒識別が壊れます）。
表示するには **表示 → 非表示シート → _students** から。

---

## よくある質問

### 生徒が氏名を変えたら？
メールアドレスで生徒を識別しているので、**氏名が変わっても同じ生徒シートに記録され続けます**。シートの A1 セルの氏名表示は次回以降の送信で最新の名前に自動更新されます。

### 同じメアドで複数の端末から学習した場合
端末を問わず同じシートに集約されます（メアドが同じであれば）。

### サマリーを毎日自動更新したい
Apps Scriptエディタで **トリガー（時計アイコン）→ トリガーを追加 → `rebuildSummaries`** を選択し、時間主導型 → 1時間ごと等を設定してください。

### 生徒情報を削除したい
1. その生徒のシートを右クリック → 削除
2. `_students` シート（非表示）から該当する行を削除
3. 必要なら「ログ」シートの該当行も削除

### URLを公開していて大丈夫？
Web AppのURLは推測困難な長い文字列で、**書き込み専用**（読み取りAPIは無し）です。万が一漏洩しても他人がデータを読めることはありませんが、念のためURLは関係者以外には共有しないでください。

### Excelに出力したい
**ファイル → ダウンロード → Microsoft Excel (.xlsx)** で書き出せます。
